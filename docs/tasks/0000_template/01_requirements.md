## 4. 非機能要件

必要に応じて `NF-XXX` の識別子を付け、実装計画書からの参照を可能にする。

- **NF-001**: `make fmt`・`make test`・`make lint` が成功する。
- **NF-002**: (性能。例: 投稿1,000件の一覧取得〜削除判定を◯秒以内に完了すること)
- **NF-003**: (セキュリティ。例: app パスワードをログ・Slack通知に含めないこと)
- **NF-004**: (信頼性・可用性。例: API呼び出し失敗時、指数バックオフで最大◯回リトライすること)
- **NF-005**: (互換性。例: Go 1.26.2 以上でビルドできること)
